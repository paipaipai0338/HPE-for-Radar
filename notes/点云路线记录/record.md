# 记录四
方法 P4

[实验目录](/experiments/P4Transformer/20260722_093849/log/log.txt)

数据 0615 - 0709

划分方式 group，每个人的数据打乱按照 7：3的比例划分

与记录三的差异：
[dataset](/data2datasets/dataset.py) 改为 [dataset_for_single](/data2datasets/dataset_for_single.py)，差异是模型不再做存在检测与多人分离，dataset_for_single 用模型的胯部中心位置进行归一化到 （0，0，0）同样这个 offset 给到点云，相当于做位置归一化，防止因为某些动作只在特定位置出现而导致模型出现位置记忆化；与此同时由于 P4 具有FPS采样，获取锚点的操作，因此利用 gt 的 3D bbox 对点云进行了筛选，防止多径等问题影响。

由于此版本不做人数估计，只做单人估计，因此忽略 BCE。

[额外实验](codex/analyze_severe_coverage.py)

严重错误样本：单个人体在单帧中的 17 个关节 MPJPE 大于 0.2m。
20cm 关节覆盖率：对于单帧人体的每个真值关节，判断其 20cm 半径范围内是否至少存在一个有效点云点；被覆盖关节数除以全部 17 个关节数。

| 单帧17关节覆盖率 | 含义 | 该组严重错误率 |
|---|---|---:|
| <25% | 最多约4个关节被覆盖 | 48.04% |
| 25%–50% | 约5–8个关节被覆盖 | 41.30% |
| 50%–75% | 约9–12个关节被覆盖 | 31.20% |
| 75%–<100% | 约13–16个关节被覆盖 | 21.50% |
| 100% | 17个关节全部被覆盖 | 9.34% |

点云稀疏或人体覆盖不完整会显著增加模型产生严重姿态估计错误的概率，是严重错误的重要来源之一；但覆盖不完整既不是严重错误的必要条件，也不是充分条件，模型泛化、姿态歧义、时序特征和网络本身的估计偏差也在起作用。

下一步考虑:
+ 对数据进行几何增广，实验验证是否泛化可进一步提升。
+ 考虑如何解决/缓解点云稀疏情况下的问题。
